All codec implementations should respond to :header with a path string identifying that codec.

mux-codec

(mux-codec & codecs)

Creates a new multiplexing codec which delegates to the given collection of codecs by reading and writing multicodec headers when coding.

By default, the first codec given is used for all encoding. The codec’s :header is written first, then the codec is used to write the value.

When decoding, the multiplexer tries to read a multicodec header and looks up the corresponding codec in the codecs map. The codec is used to read a value from the remaining data.

As a consequence, the delegated codecs must not write or expect to consume their own headers!

wrap-headers

(wrap-headers codec)(wrap-headers codec header)

Creates a new codec which will write the header for the wrapped codec before calling it. When decoding, it will read the header before calling the wrapped codec, and throw an exception if it does not match.
